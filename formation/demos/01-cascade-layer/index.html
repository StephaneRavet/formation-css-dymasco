Démo cascade @layer — sanity check formateur

Ouvre chaque étape, vérifie que le rendu match l'attendu, puis passe à la suivante. DevTools panneau Styles ouvert pour voir les couches.

  1. Étape 1 — Tentative naïve Attendu : titre cards noir (override KO). Spé 0,5,0 vs 0,1,0 = Apriso gagne.
  2. Étape 2 — @layer seul, le piège Attendu : titre cards toujours noir. Apriso unlayered bat @layer overrides.
  3. Étape 3 — @import url() layer() Attendu : titre cards rouge ✅. Apriso wrappé dans framework, overrides gagne. Background alert encore jaune (le !important framework tient).
  4. Étape 4 — Battre les !important via couche priority Attendu : titre rouge ✅ + background alert vert clair ✅. priority déclarée avant framework → ordre inversé pour !important.
  5. Étape 5 — Tokens CSS Attendu : identique étape 4, mais piloté par variables --ml-*. Changer une variable → tout suit.

Points à vérifier dans DevTools (étapes 3-5) :

  • Network : apriso-mini.css chargé une seule fois (via @import, pas via <link>).
  • Styles : colonne Layer visible sur chaque déclaration.
  • Computed : la source d'une valeur indique le nom de la couche.
